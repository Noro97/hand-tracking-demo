### 2026-05-29 — `gh repo create --source=. --remote=origin --push` за один шаг
**From:** задача 012 (rated 5/5)
**Lesson:** Когда нужно создать remote-репозиторий из локального проекта — `gh repo create <user>/<name> --public --source=. --remote=origin --push --description="..."` делает всё атомарно: создаёт репо на GitHub, добавляет remote, привязывает branch tracking, пушит. Альтернатива (`gh repo create` без `--source`, потом `git remote add`, потом `git push -u origin master`) — три раздельных шага, каждый из которых может тихо упасть.

### 2026-05-29 — Strict TypeScript — это fix-cascade, не точечная правка
**From:** задача 011 (rated 5/5)
**Lesson:** Включение `strict: true` (и тем более `noUncheckedIndexedAccess`) в проекте, где их не было с начала, **никогда не получится сделать одной строкой в `tsconfig.json`**. Surface'тся: отсутствующие `@types/*` пакеты (которые маскировались `skipLibCheck`), implicit `any` в callback'ах, доступы к массивам без guards, `useRef<T>(null)` сигнатуры в strict-режиме. Реалистичная оценка: 5–15 точек правок на средний проект. Включать strict нужно **вместе с готовностью разгрести cascade одним проходом**, а не «попробуем включить».

### 2026-05-29 — Scale-invariant жесты через relative thresholds
**From:** задача 009 (rated 5/5)
**Lesson:** Для webcam-жестов пороги обязаны быть **относительными** к размеру руки в кадре, не абсолютными в пикселях/нормализованных координатах. Иначе тот же физический жест даёт разные значения метрики на разных дистанциях руки до камеры (close = большие, far = маленькие) и жест ломается. Reference-длина: расстояние от запястья (landmark 0) до middle-MCP (landmark 9) — это «скелетный масштаб» руки, инвариантный к ракурсу. Применять для любых будущих жестов (fist-closure, swipe-distance, finger-spread).

### 2026-05-29 — `preview_snapshot` как backup для `preview_screenshot`
**From:** задача 008 (rated 5/5)
**Lesson:** Когда `preview_screenshot` MCP-инструмент таймаутит (флаки рендерер, навигация залипла), переходить на `preview_snapshot` — он строит accessibility-tree через другой code path и обычно работает. Бонус: a11y-tree показывает ВСЕ элементы DOM независимо от z-index/видимости, что иногда полезнее, чем скриншот (например, для проверки HUD'а, который перекрыт loading-overlay'ем).

### 2026-05-29 — git config-блок проверять до `git add`
**From:** задача 007 (rated 5/5)
**Lesson:** Перед задачами на первичный `git commit` (после `git init` без существующей истории) — заранее проверять `git config --get user.email`. Если пусто, поднимать вопрос на этапе предложения задачи, а не упираться в `fatal: unable to auto-detect email` при выполнении. Safety-protocol запрещает агенту править `git config`, поэтому блок ВСЕГДА требует пользовательского ввода. Лучше выяснить это до того, как stage'нули 25 файлов.

### 2026-05-29 — Конкретность в стратегических брифах
**From:** задача 004 (rated 5/5)
**Lesson:** В стратегических документах MVP-планы и success criteria должны быть привязаны к **именованным валидированным артефактам** домена (конкретный clinical test типа BBT, конкретные CPT-коды, конкретные EHR-системы, конкретные конкуренты с ценами), а не к абстрактным «оценкам прогресса» и «возмещению». Это превращает документ из описательного в actionable и сразу выявляет дыры в знаниях о домене.

### 2026-05-29 — Троттлинг setState из MediaPipe-loop
**From:** задача 003 (rated 5/5)
**Lesson:** `hands.onResults` вызывается на каждом кадре камеры (~30 Гц). Если зеркалить состояние в React через `useState` напрямую — будет 30 ре-рендеров в секунду, а данные для HUD меняются раз в секунды. Паттерн: держать «горячее» состояние в `useRef` для отрисовки на canvas; «холодное» зеркало для React-HUD обновлять через ref-таймер (~10 Гц достаточно для UI). Применять везде, где UI-индикатор берёт данные из MediaPipe/getUserMedia-цикла.

### 2026-05-29 — Порядок при удалении импортируемого файла
**From:** задача 002 (rated 5/5)
**Lesson:** При удалении файла, который импортируется другими модулями, *сначала* очистить импорты у потребителей, *потом* удалять файл. Если сделать наоборот, Vite (любой HMR) пытается транспилировать stale-импорт между правками и засоряет логи `Internal server error: Failed to resolve import "..."`. Сборка может пройти, потому что финальное состояние корректно, но dev-сервер показывает ложные ошибки, которые легко принять за реальные.

### 2026-05-29 — Минимальная prep-зачистка перед рефакторами
**From:** задача 001 (rated 5/5)
**Lesson:** Когда пользователь просит «cleanup → рефактор» в одной директиве, делать prep-зачистку отдельной мелкой задачей с *честной* минимальной областью (только то, что реально мертво сегодня, без захвата того, что снимется в 002/003). Это даёт чистый чекпоинт и не дублирует работу.
