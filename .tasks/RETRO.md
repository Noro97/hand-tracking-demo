### 2026-05-29 — Конкретность в стратегических брифах
**From:** задача 004 (rated 5/5)
**Lesson:** В стратегических документах MVP-планы и success criteria должны быть привязаны к **именованным валидированным артефактам** домена (конкретный clinical test типа BBT, конкретные CPT-коды, конкретные EHR-системы, конкретные конкуренты с ценами), а не к абстрактным «оценкам прогресса» и «возмещению». Это превращает документ из описательного в actionable и сразу выявляет дыры в знаниях о домене.

### 2026-05-29 — Троттлинг setState из MediaPipe-loop
**From:** задача 003 (rated 5/5)
**Lesson:** `hands.onResults` вызывается на каждом кадре камеры (~30 Гц). Если зеркалить состояние в React через `useState` напрямую — будет 30 ре-рендеров в секунду, а данные для HUD меняются раз в секунды. Паттерн: держать «горячее» состояние в `useRef` для отрисовки на canvas; «холодное» зеркало для React-HUD обновлять через ref-таймер (~10 Гц достаточно для UI). Применять везде, где UI-индикатор берёт данные из MediaPipe/getUserMedia-цикла.

### 2026-05-29 — Порядок при удалении импортируемого файла
**From:** задача 002 (rated 5/5)
**Lesson:** При удалении файла, который импортируется другими модулями, *сначала* очистить импорты у потребителей, *потом* удалять файл. Если сделать наоборот, Vite (любой HMR) пытается транспилировать stale-импорт между правками и засоряет логи `Internal server error: Failed to resolve import "..."`. Сборка может пройти, потому что финальное состояние корректно, но dev-сервер показывает ложные ошибки, которые легко принять за реальные.

### 2026-05-29 — Минимальная prep-зачистка перед рефакторами
**From:** задача 001 (rated 5/5)
**Lesson:** Когда пользователь просит «cleanup → рефактор» в одной директиве, делать prep-зачистку отдельной мелкой задачей с *честной* минимальной областью (только то, что реально мертво сегодня, без захвата того, что снимется в 002/003). Это даёт чистый чекпоинт и не дублирует работу.
